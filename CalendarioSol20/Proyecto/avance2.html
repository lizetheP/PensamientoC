Avance 2. Modelando las entradas y salidas de mi proyecto 
.
.

Objetivo:

Modelar las entradas y salidas de tu proyecto final.


Forma de trabajo:

Actividad colaborativa


   Instrucciones:
 
      En esta actividad se hace una modelación de las especificaciones de los tipos de datos y formatos para las entradas y salidas del proyecto. Es decir, aquí se define si el programa lee de consola o de archivo de texto, si despliega un resultado en pantalla o lo guarda entre uno o más archivos. Aquí se define la interacción que se espera que el usuario y programa tengan de manera específica.
       
    • Definir la interacción que se espera que el usuario y programa tengan de manera específica: 
      • Definir cómo se capturan las entradas, qué tipos de datos son requeridos, cuáles son los datos que su programa va a requerir y de donde va a salir cada uno de ellos, es decir cuáles los proporciona el usuario y cuales los calculan o ya los tienen almacenados.
      • Definir cómo se despliegan las salidas, que tipos de datos son, en donde se van a desplegar, el formato que van a tener etc. 
    • Definir si el programa lee de consola y/o de archivo de texto, si despliega un resultado en pantalla y/o lo guarda entre uno o más archivos.
